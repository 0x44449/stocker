# 뉴스 크롤러 스케줄러에 락 적용

## 배경

- 현재 NewsCrawlLock이 API 엔드포인트에서만 사용됨
- 스케줄러는 락 없이 실행되어 API와 스케줄러 간 중복 실행 가능
- API 크롤링 중 스케줄러가 실행되면 같은 프로바이더가 동시에 크롤링됨

## 목표

스케줄러에도 락을 적용하여 API ↔ 스케줄러 간 중복 실행 방지

## 관련 파일

```
apps/api/src/main/java/com/hanzi/stocker/ingest/news/NewsCrawlScheduler.java
apps/api/src/main/java/com/hanzi/stocker/ingest/news/NewsCrawlLock.java
```

## 할 것

NewsCrawlScheduler에서 engine.crawl() 호출 전후로 락 적용:
- tryLock 실패 시 해당 프로바이더 스킵
- 크롤링 완료 후 unlock (finally)

## 안 할 것

- NewsCrawlLock 구조 변경
- API 엔드포인트 수정
