# 뉴스 크롤러 로그 추가

## 배경

- 현재 뉴스 크롤러가 스케줄러(`@Scheduled`)로 동작하지만 로그가 없어서 실행 여부 확인이 불가
- Docker 환경에서 스케줄러가 정상 동작하는지 확인 필요

## 목표

크롤러 실행 여부와 진행상황을 파악할 수 있도록 로그 추가

## 관련 파일

```
apps/api/src/main/java/com/hanzi/stocker/ingest/news/
├── NewsCrawlScheduler.java  # 스케줄러 - 로그 추가
├── NewsCrawlEngine.java     # 크롤링 엔진 - 로그 추가
```

## 할 것

### NewsCrawlScheduler
- INFO: 스케줄러 시작/종료

### NewsCrawlEngine
- INFO: 진행상황 파악 수준 (기사 URL 수집 개수, 저장 건수 등)
- DEBUG: 상세 (개별 기사 처리, 스킵 사유 등)

## 안 할 것

- 엔드포인트 추가 (별도 작업)
- KRX 크롤러 로그 (별도 작업)
