# KrxMasterCrawlEngine 액면가 파싱 수정

## 배경

- 종목 마스터 CSV 다운로드 성공
- 파싱 중 `액면가` 컬럼에 "무액면" 텍스트가 있어서 NumberFormatException 발생
- 투자회사/펀드 등은 액면가가 없어서 "무액면"으로 표기됨

## 목표

액면가 컬럼만 숫자가 아닌 값을 null로 처리 (다른 컬럼은 엄격하게 유지)

## 관련 파일

```
apps/api/src/main/java/com/hanzi/stocker/ingest/krx/master/KrxMasterCrawlEngine.java
```

## 할 것

### 1. parseLongOrNull 메서드 추가

```java
/**
 * 숫자가 아닌 값은 null로 처리하는 느슨한 버전.
 * 특수 케이스(예: 액면가의 "무액면")에만 사용.
 */
private Long parseLongOrNull(String value) {
    if (value == null || value.isBlank()) {
        return null;
    }
    try {
        return Long.parseLong(value.replace(",", ""));
    } catch (NumberFormatException e) {
        return null;
    }
}
```

### 2. 액면가 파싱 시 parseLongOrNull 사용 + 주석

```java
entities.add(new StockMasterEntity(
    ...
    blankToNull(record.get("소속부")),
    record.get("주식종류"),
    // 액면가: 투자회사/펀드 등은 "무액면"으로 표기되어 숫자가 아닌 값은 null 처리
    parseLongOrNull(record.get("액면가")),
    parseLong(record.get("상장주식수"))
));
```

### 3. 기존 parseLong은 그대로 유지 (엄격한 버전)

## 완료 조건

- "무액면" 등 숫자가 아닌 액면가 값은 null로 저장
- 상장주식수 등 다른 숫자 컬럼은 여전히 엄격하게 검증
- 크롤링 정상 완료
