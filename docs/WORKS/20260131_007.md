# deploy.sh 기본 배포 대상에서 postgres 제외

## 배경

- 현재 `./deploy.sh` 실행 시 postgres 포함 전체 서비스가 재시작됨
- postgres는 한번 띄워두면 거의 안 건드리는데 불필요하게 재시작됨
- 앱(api, analyzer)만 배포하고 싶은 경우가 대부분

## 목표

기본 배포 대상을 api, analyzer로 변경. postgres는 명시적으로 지정할 때만 배포.

## 관련 파일

```
deploy.sh
```

## 할 것

### 1. 서비스 미지정 시 기본값 변경

현재: 전체 서비스 (postgres, api, analyzer)
변경: api, analyzer만

### 2. --down 동작 변경

현재: `docker-compose down` (전체)
변경: 지정된 서비스만 stop/rm

### 3. 사용법 안내 수정

```
Examples:
  $0                      # api, analyzer 배포
  $0 postgres             # PostgreSQL만 배포
  $0 postgres api analyzer  # 전체 배포
```

## 안 할 것

- docker-compose.yml 수정
- 프로필 분리
