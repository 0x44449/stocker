# Expo 모바일 클라이언트 초기 세팅

## 배경

stocker 피드 UI 프로토타입(docs/prototypes/feed-v4.jsx)을 실기기에서 확인하기 위해 Expo 프로젝트를 세팅한다.
mock 데이터로 피드를 채워두고, 실제 폰에서 스크롤하며 카드 크기·터치 영역·UX를 검증한다.
이후 "이 mock을 real로 바꾸려면 뭐가 필요한가"를 역으로 도출하여 파이프라인 요구사항을 정리하는 것이 목적이다.

## 목표

Expo 프로젝트를 세팅하고, 피드 화면 하나를 mock 데이터로 띄운다.

## 할 것

- `apps/stocker-mobile`에 Expo 프로젝트 생성
- 피드 화면 구현 (feed-v4.jsx 기반)
  - 이벤트 카드
  - 섹터 분위기 카드
  - 과거 비교 카드
  - 이상 신호 카드
  - 종목 연결 카드
  - 통계 카드
  - 장 구간 구분선 (장전/장중/장후)
  - 시간 표시 (장전 07:32 형태)
  - 커뮤니티 영역 (투표, 리액션, 인기 댓글)
- mock 데이터는 feed-v4.jsx의 FEED 배열 기반
- Expo Go로 실기기에서 확인 가능한 상태

## 안 할 것

- 관심종목/설정 등 다른 탭 화면
- 네비게이션 구조 (탭 바 포함, 피드 한 화면만)
- API 연동
- 상태 관리 라이브러리 도입
- 카드 탭 시 상세 화면

## 참고

- UI 프로토타입: `docs/prototypes/feed-v4.jsx`
- 이 파일은 웹용 React(inline style)로 작성된 디자인 레퍼런스이다
- 카드 타입, 데이터 구조, 시각적 방향은 이 파일을 따르되, 웹 코드를 그대로 이식하지 않는다
- React Native에 맞는 컴포넌트와 스타일로 새로 구현한다
