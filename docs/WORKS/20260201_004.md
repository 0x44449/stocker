# KRX 종목 마스터 크롤러 로그 추가

## 배경

- POST /internal/crawl/stock-master 호출 시 DB에 데이터가 안 채워짐
- 비동기 실행이라 에러가 삼켜지고, 엔진에 로그도 부족해서 원인 파악 불가

## 목표

KrxMasterCrawlEngine에 상세 로그 추가하여 문제 발생 시 원인 파악 가능하게 함

## 관련 파일

```
apps/api/src/main/java/com/hanzi/stocker/ingest/krx/master/KrxMasterCrawlEngine.java
```

## 할 것

### 로그 추가 위치

1. 크롤링 시작
2. CSV 다운로드 완료 (바이트 수)
3. 파싱 완료 (건수)
4. 저장 완료
5. 에러 발생 시 (try-catch)

### 예시

```java
public void crawl() {
    log.info("종목 마스터 크롤링 시작");
    
    // ... formData 설정
    
    var csvBytes = fileClient.download(...);
    log.info("CSV 다운로드 완료 - {} bytes", csvBytes.length);
    
    try (...) {
        // 파싱
        log.info("CSV 파싱 완료 - {}건", entities.size());
        
        repository.saveAll(entities);
        log.info("종목 마스터 크롤링 완료 - {}건 저장", entities.size());
    } catch (Exception e) {
        log.error("종목 마스터 크롤링 실패", e);
        throw e;
    }
}
```

## 완료 조건

- 크롤링 각 단계별 로그 확인 가능
- 에러 발생 시 스택트레이스 로그 확인 가능
