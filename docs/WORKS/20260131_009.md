# Health, NewsCrawl 컨트롤러 Swagger 응답 정보 추가

## 배경

- 현재 컨트롤러들이 `Map<String, String>`을 반환해서 Swagger에서 실제 응답 필드를 알 수 없음
- API 문서화를 위해 응답 타입 정보 필요

## 목표

HealthController, NewsCrawlController에 Swagger 응답 정보 추가

## 관련 파일

```
apps/api/src/main/java/com/hanzi/stocker/api/HealthController.java
apps/api/src/main/java/com/hanzi/stocker/api/internal/NewsCrawlController.java
```

## 할 것

### 1. HealthController

현재:
```java
public Map<String, String> health() {
    return Map.of("status", "ok");
}
```

변경:
- HealthResponse DTO 생성 (status 필드)
- 반환 타입 변경

### 2. NewsCrawlController

현재:
```java
// trigger
public Map<String, String> trigger(...) {
    return Map.of("status", "started", "provider", providerId);
    // 또는 "status": "already_running" / "not_found"
}

// status
public Map<String, String> status() {
    return crawlLock.allStatus(...);  // {"hk": "running", "mk": "idle"} 형태
}
```

변경:
- CrawlTriggerResponse DTO 생성 (status, provider 필드)
- trigger 메서드 반환 타입 변경
- status 메서드는 프로바이더 목록이 동적이므로 어노테이션으로 예시 추가

## 방침

- 가급적 DTO 사용
- 동적 키 등 DTO로 표현 어려운 경우 @ApiResponse 어노테이션으로 예시 추가

## 안 할 것

- 미구현 컨트롤러(Stock, Watchlist) 수정
- 에러 응답 스키마 정의
